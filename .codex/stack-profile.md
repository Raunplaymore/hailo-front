# Stack Profile: Web - Vite React

Generated by `setup-codex-agent-crews`. Read this before changing Vite React projects.

## Stack

- Framework: Vite
- UI: React 18+ or 19+
- Language: TypeScript when `tsconfig.json` exists
- Routing: React Router when installed
- Styling: Tailwind, CSS Modules, or project-local CSS; follow existing code first
- Build: `vite build` or project `build` script

## Path Mapping

| Type | Location |
|---|---|
| App entry | `src/main.tsx` or `src/main.jsx` |
| Root component | `src/App.tsx` or `src/App.jsx` |
| Pages/routes | `src/pages/` or route modules used by the project |
| Components | `src/components/` |
| Hooks | `src/hooks/` |
| API client | `src/api/`, `src/lib/api.ts`, or existing data layer |
| Types | `src/types/` or colocated `*.types.ts` |
| Styles | `src/index.css`, `src/app.css`, CSS modules, or Tailwind config |

## Naming

- Components: PascalCase.
- Hooks: `useName`.
- Route files: match the project convention.
- Prefer named exports unless the project consistently uses defaults.

## Rules

- Follow the existing folder style before adding new architecture.
- Keep browser-only code out of module initialization when it breaks SSR-like test runners.
- Do not introduce a new state library when local state or the existing store is enough.
- Keep environment variables under Vite's public convention (`VITE_*`) and never expose secrets.
- Preserve accessibility basics: labels, keyboard operation, focus states.

## High-Risk Areas

- Router setup and route guards.
- Build config (`vite.config.*`, Tailwind/PostCSS config).
- Auth/session handling.
- API base URL and environment variable handling.

## Initial Scan

- `src/`
- `src/components/`
- `src/pages/` or router files
- `vite.config.*`
- `package.json`
